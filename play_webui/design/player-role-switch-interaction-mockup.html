交互规则

核心
  • 1入口放在 SessionRoom 右上“设置”菜单，不新增顶栏独立按钮。
  • 2角色列表只来自当前 story mounted characters，且是单选。
  • 3选择当前角色时确认按钮禁用；选择不同角色后需要二次确认。
  • 4切换不重写历史消息，只影响后续 user 身份和后续 prompt 语义。
  • 5WebUI 面板选择与 /role_bind 命令共用后端绑定/切换能力。

渠道协议

共用逻辑
入口设置菜单 -> 当前扮演角色 -> 切换
动作选择角色后调用 player role bind/switch 能力
异常缺失、未挂载、角色删除时强制进入选择弹窗
结果更新 session 绑定和 snapshot，刷新 SessionRoom 展示

切换扮演角色

选择一个当前故事已挂载角色。切换不会改写已有历史，assistant 仍显示为旁白。

这个选择与文本渠道 /role_bind 序号 使用同一条后端能力；WebUI 只是把序号选择换成了角色卡选择。

确认切换玩家角色

这会改变后续 user 消息的身份展示和后续 prompt 语义。
